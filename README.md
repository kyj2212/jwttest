## JWT 테스트

#### 여러가지 인증 방식
    - 쿠키 방식
    - 세션 방식
    - 세션 서버 방식 (with redis)
    - JWT (토큰 방식)

#### JWT 방식의 장점과 단점

    - 장점 : 별도의 인증저장소가 필요없다.
             쿠키로부터 발생하는 취약점의 대안책, 트래픽에 대한 부담이 낮음, REST 서비스로 제공 가능
    - 단점 : 토큰 자체에 정보를 담고 있다. 보안취약점이 될수 있다.
             payload 인코딩의 취약, 토큰 길이가 길어질수록 네트워크에 부하를 줄수 있다.
            stateless 이기 때문에 토큰 만료시간이 반드시 필요
